# Personal Dev Hub Builder

## Purpose
Guide the development of a static personal developer website with biodata, portfolio, services, developer documentation, and GitHub Pages deployment.

## Core Rules
- Always keep the project static-first.
- Do not add backend, database, login, admin panel, CMS, or payment features unless explicitly requested.
- Prioritize maintainable Astro structure, reusable components, responsive UI, and GitHub Pages compatibility.
- Keep the visual direction professional: modern AI wireframe UI with subtle RPG/pixel accents.
- Before adding features, check whether they belong to the MVP scope.

## Workflow
1. Read `references/project-brief.md` to understand the project goal.
2. Read `references/feature-scope.md` before planning or adding features.
3. Read `references/design-direction.md` before UI or visual implementation.
4. Read `references/development-rules.md` before coding.
5. Review whether the request fits the MVP scope.
6. Suggest a maintainable Astro structure or implementation plan.
7. Implement using reusable components and static content.
8. Validate responsiveness, readability, performance, and GitHub Pages compatibility.
