# Design Direction

## Main Design Theme
Modern AI Developer Interface with subtle 2D RPG / pixel game inspiration.

The website should look professional first, then playful as a secondary identity.

Recommended balance:
- 80% modern professional UI
- 20% RPG/pixel/gamification accent

## Design Mood
The design should feel:
- Modern
- Futuristic
- Professional
- Clean
- Slightly playful
- Developer-focused
- Gamified but not childish
- Dark and atmospheric

## Visual Keywords
Use these keywords as design guidance:
- AI wireframe
- Cyber grid
- Neon glow
- Floating panels
- Glassmorphism
- Pixel RPG frame
- Quest log
- Skill tree
- Developer dashboard
- Digital workshop
- Classical RPG interface

## Color Direction
Recommended color palette:
- Main background: near black, dark navy, or deep slate
- Primary accent: cyan, blue, or electric teal
- Secondary accent: purple, emerald, or soft green
- Text primary: white or near white
- Text secondary: slate gray or muted blue-gray
- Border: soft neon line or translucent border
- Card background: dark translucent panel

Avoid:
- Too many bright colors
- Full rainbow neon
- Low contrast text
- Pure black and pure white combination if it feels harsh

## Layout Direction
Use a clean section-based landing page layout:
1. Navbar
2. Hero section
3. Profile summary
4. Featured projects
5. Services preview
6. Dev Notes preview
7. Contact/footer

Each section should have enough spacing and should not feel crowded.

## Component Style
Recommended component style:
- Rounded cards
- Soft shadow
- Thin glowing border
- Hover animation
- Gradient background
- Small pixel ornament
- Terminal-like code blocks
- RPG-style status panels
- Skill badges
- Quest log cards

## RPG / Gamification Elements
Use RPG concepts as metaphors, not as the main visual system.

Possible elements:
- Skill tree for technical skills
- Quest log for current projects or learning roadmap
- Achievement badge for milestones
- Character profile card for biodata
- Inventory-style card for tools and stack
- Pixel border accents
- Small pixel icons or decorative sprites

## AI / Futuristic Elements
Possible elements:
- Wireframe grid background
- Node connection lines
- Floating orb or gradient glow
- Animated background lines
- Code/terminal visual
- Circuit pattern
- Subtle scanning line

## Animation Direction
Animations should be smooth and subtle.

Recommended:
- Fade in on scroll
- Floating background objects
- Hover lift on cards
- Soft glowing border
- Smooth page transition
- Skill bar animation
- Button hover effect

Avoid:
- Too many moving objects
- Fast or distracting animation
- Heavy animation that hurts performance
- Animation that makes text hard to read

## Typography
Use modern readable fonts for most content.

Recommended:
- Main font: modern sans-serif
- Code font: monospace
- Pixel font: only for small accents, headings, badges, or decorative labels

Do not use pixel font for long paragraphs because readability will suffer.

## UI Rules
1. Prioritize readability.
2. Keep design professional.
3. Use RPG/pixel elements as accents only.
4. Maintain consistent spacing.
5. Make mobile layout clean and simple.
6. Use reusable components.
7. Avoid visual clutter.
8. Do not sacrifice performance for animation.
9. Do not make it look like a full game website.
10. The website must still be appropriate for clients and recruiters.

## Visual Identity Summary
The final design should feel like:

"A professional developer portfolio that looks like a futuristic command center with subtle RPG adventure elements."
