# Project Brief - Personal Developer Hub

## Project Name
Personal Developer Hub / Journey Camp

## Project Goal
Membangun website pribadi berbasis static site yang berfungsi sebagai identitas digital, portfolio, landing page jasa pembuatan sistem, dan dokumentasi command/glosarium teknologi yang sering digunakan.

Website ini akan dideploy menggunakan GitHub Pages dan harus tetap ringan, responsive, profesional, serta mudah dikembangkan.

## Core Concept
Website ini bukan hanya portfolio biasa, tetapi menjadi pusat identitas digital pribadi yang mencakup:
- Biodata personal
- Portfolio project
- Promosi jasa pembuatan sistem
- Dokumentasi command dan glosarium teknologi
- Catatan perjalanan belajar sebagai developer

## Target Users
1. Calon klien yang ingin melihat jasa pembuatan sistem.
2. Recruiter, dosen, atau pihak akademik yang ingin melihat kemampuan dan portfolio.
3. Teman, mahasiswa, atau developer pemula yang ingin membaca dokumentasi command.
4. Pemilik website sendiri sebagai tempat menyimpan catatan teknis pribadi.

## Platform
Website static yang berjalan di GitHub Pages.

## Main Stack
- Astro
- Tailwind CSS
- TypeScript
- Markdown / MDX
- GitHub Pages

## MVP Pages
1. Home
2. About
3. Portfolio
4. Services
5. Dev Notes
6. Contact

## MVP Features

### Home
- Hero section
- Short personal tagline
- CTA ke portfolio dan services
- Preview portfolio
- Preview dev notes
- Visual modern AI + RPG/pixel accent

### About
- Biodata pribadi
- Developer profile
- Skill overview
- Tech stack
- Learning journey

### Portfolio
- List project
- Detail project
- Project description
- Problem solved
- Features
- Tech stack
- Screenshot or preview
- Link demo/repository if available

### Services
- List jasa pembuatan sistem
- Service category
- Explanation of what user can request
- CTA contact via email/WhatsApp/social link

### Dev Notes
- Documentation for frequently used commands and concepts
- Categories:
  - Laravel
  - Git
  - Docker
  - Flutter
  - MySQL
  - Terminal
  - Troubleshooting
- Each note should explain command, function, when to use it, and example usage.

### Contact
- Email
- GitHub
- LinkedIn
- WhatsApp or other contact link if available

## Content Strategy
Use static content files instead of backend or database.

Recommended content sources:
- Portfolio: Markdown/MDX collection or TypeScript data
- Services: TypeScript data
- Skills: TypeScript data
- Dev Notes: Markdown/MDX
- Profile: TypeScript data

## Non-Goals for MVP
Do not build these in the first version:
- Login
- Database
- Backend API
- Admin dashboard
- Payment system
- CMS integration
- Comment system
- User accounts
- Complex analytics
- Real-time features

## Success Criteria
The MVP is considered successful when:
1. Website can be deployed and opened through GitHub Pages.
2. Website is responsive on desktop, tablet, and mobile.
3. All MVP pages are available.
4. Website has consistent visual identity.
5. Portfolio section contains at least 3 projects.
6. Dev Notes section contains at least 5 notes or commands.
7. Services section clearly explains what services are offered.
8. Website feels modern and professional while still having subtle RPG/pixel identity.
